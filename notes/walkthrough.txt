ROOT CAUSE NOTE: KEYWORD WEIGHTS DATA QUALITY
======================================================================

The core issue (H1, H2) is that precalc rag_examples.keyword_weights
contains three categories of data:

  Type A [CORRECT]: learn_keywords IDs
    {"negative_exponents": 0.28, "product_rule_for_exponents": 0.3}
    These work correctly — strength tracking and "Focusing on" both right.

  Type B [WRONG]: legacy AP Calc keyword IDs
    {"u_substitution": 0.44, "definite_integral": 0.26}
    Strength tracking updates with wrong keys; "Focusing on" shows
    AP Calc concept labels on precalc problems.

  Type C [EMPTY]: {}
    strength tracking is skipped entirely.

Fix: Run the keyword tagging tool in admin (/api/learn/retag-with-keyword
or the /keywords admin page) on ALL precalc rag_examples to replace
Type B and Type C with correct learn_keywords IDs.
