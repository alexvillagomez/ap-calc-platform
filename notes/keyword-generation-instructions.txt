PRECALC KEYWORD CATALOG — PROJECT BRIEF
========================================

GOAL
----
Build a ~1000 topic keyword catalog for a precalculus adaptive learning platform.
Keywords are used to tag problems via cosine similarity: a problem description is
embedded and matched against keyword embeddings. Description and example quality
directly determines tagging accuracy.


STRUCTURE
---------
Two levels: UMBRELLA → TOPIC

UMBRELLA keywords group related topic keywords within a category. A student's umbrella
score = average of its children's scores. Umbrellas need a description only — no examples.

TOPIC keywords are specific learnable skills. Each belongs to one umbrella.
Each topic keyword needs four fields:

  id           snake_case, specific. "factoring_trinomials_leading_coefficient_1" not "factoring"
  label        Human-readable. "Factoring trinomials (a = 1)"
  description  1–2 sentences. Precise enough to distinguish from neighboring keywords.
               Name what the student does, what mathematical form is involved, and
               what makes this different from similar skills.
  examples     3–5 complete problem stems. EVERY example must start with an action verb.
               "Factor x² + 5x + 6" not "x² + 5x + 6"
               The verb is what separates factoring from evaluating, solving, or graphing
               the same expression.


OUTPUT FORMAT
-------------
JSON, one category per message (catalog is too large for one response).

{
  "category_id": "polynomials",
  "category_label": "Polynomials",
  "umbrellas": [
    {
      "id": "polynomial_factoring",
      "label": "Polynomial Factoring",
      "description": "Rewriting polynomial expressions as products of simpler factors.",
      "keywords": [
        {
          "id": "factoring_trinomials_leading_coefficient_1",
          "label": "Factoring trinomials (a = 1)",
          "description": "Factor a monic quadratic x² + bx + c into (x + p)(x + q) by finding integers p, q where p·q = c and p + q = b.",
          "examples": [
            "Factor x² + 7x + 12.",
            "Factor x² - 5x + 6.",
            "Factor x² + x - 20.",
            "Write x² - 3x - 18 as a product of two binomials."
          ]
        }
      ]
    }
  ]
}


GRANULARITY RULE
----------------
One keyword = one specific teachable sub-skill.

Too broad → split:   "factoring" → GCF / trinomials a=1 / trinomials a≠1 /
                     difference of squares / sum of cubes / by grouping
Too narrow → merge:  "factoring x² + 5x + 6" is a problem, not a skill
