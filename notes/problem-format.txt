=============================================
  PROBLEM GENERATION FORMAT
=============================================

FORMATTING RULES:
  - Plain text for prose — no $ needed
  - $...$ for inline math (variables, expressions, integrals)
  - $$...$$ for standalone equations (own line, display size)
  - NEVER use \text{} or \displaystyle
  - Inline integrals: $\int_0^2 f(x)\,dx$
  - Standalone integrals: $$\int_0^2 f(x)\,dx$$

=============================================
  WORKFLOW
=============================================

  STEP 1 — CREATION
  Write the problem with all four keyword descriptions.
  Leave the weight fields out — they are auto-populated.

  STEP 2 — AUTO-TAGGING (on DB insert)
  When the problem enters the DB (via RAG Examples or rag-agent),
  each description is embedded and matched against the keyword catalog
  using cosine similarity:
    topic_description       → keyword_weights   (LLM rerank, top-20 candidates)
    action_description      → action_weights    (cosine-only, top-4 above 0.35)
    representation_description → representation_weights (cosine-only, top-3)
    prerequisite_description → prerequisite_weights (cosine-only, top-5)

  STEP 3 — MANUAL REVIEW (optional, in Preview JSON)
  Use the four keyword tabs to inspect or override the auto-tagged weights.

=============================================
  FIELDS
=============================================

  problem_description        What skill(s) this problem tests. 1–2 sentences, no LaTeX.

  latex_content              Problem stem. Prose as plain text, math in $...$.

  choices                    MCQ only. Exactly 4 strings, each wrapped in $...$.

  correct_index              MCQ only. Integer 0–3.

  wrong_answer_data          MCQ only. One entry per wrong choice:
                               { "index": <int>, "description": "<student error>" }
                             Describe the specific mistake, not just "wrong answer."

  solution_latex             Full worked solution. Same formatting as latex_content.
                             Use $$\begin{aligned}...\end{aligned}$$ for step-by-step work.

  rubric                     FRQ only. One line per point, plain text + $math$.

  difficulty                 Integer 1–5.
                               1 = single step, direct formula
                               2 = two or three steps, clean arithmetic
                               3 = standard AP level, moderate chaining
                               4 = challenging, requires insight or multi-concept chain
                               5 = very hard, non-obvious or dense multi-step

  ── KEYWORD DESCRIPTION FIELDS (written at creation time) ───────────────────

  topic_description          1–2 sentences: what topic/skill this problem tests.
                             Embedded and matched against learn_keywords → keyword_weights.

  action_description         1–2 sentences: what the student must do (the cognitive verb).
                             Embedded and matched against action_items keywords → action_weights.
                             e.g. "Students must differentiate and analyze sign behavior."

  representation_description 1–2 sentences: how the problem is presented (format/medium).
                             Embedded and matched against representations keywords → representation_weights.
                             e.g. "The function is given as an algebraic equation."

  prerequisite_description   1–2 sentences: prior knowledge required to solve this.
                             Embedded and matched against topic keywords → prerequisite_weights.
                             e.g. "Requires the power rule and factoring quadratics."

  ── KEYWORD WEIGHT FIELDS (auto-populated on DB insert, editable in Preview JSON) ──

  keyword_weights            Cosine + LLM rerank of topic_description against learn_keywords.
  action_weights             Cosine match of action_description against action_items keywords.
  representation_weights     Cosine match of representation_description against representations.
  prerequisite_weights       Cosine match of prerequisite_description against topic keywords.
